# Icons

## Overview

Axis uses Lucide for product UI icons. Lucide provides the SVG shapes; Axis tokens define how icons appear in the interface.

<Canvas of={Stories.TokenStyledIcons} />

## Usage

- Import only the icons you need from `@lucide/vue`.
- Let icons inherit text color with `currentColor` unless the icon communicates a semantic state.
- Size icons deliberately with Axis component or semantic sizing tokens.
- Keep stroke width consistent within a component family.
- Use one icon style in product UI. Do not mix Lucide, PrimeIcons, Material Symbols, and custom SVGs in the same surface unless the design system documents an exception.

```vue
<script setup lang="ts">
import { Search } from '@lucide/vue'
</script>

<template>
  <Search
    aria-hidden="true"
    :size="20"
    :stroke-width="2"
    class="axis-search-icon"
  />
</template>

<style scoped>
.axis-search-icon {
  color: var(--axis-color-text-secondary);
}
</style>
```

## Token Behavior

Lucide icons should follow the surrounding component by default:

| Property | Axis rule |
|---|---|
| Color | Use `currentColor` or a semantic color token such as `--axis-color-text-secondary`, `--axis-color-interactive-primary`, or `--axis-color-feedback-error`. |
| Size | Use a component token when the icon belongs to a component, or a documented semantic size when used standalone. |
| Stroke | Keep a consistent stroke width for the component context. Use Lucide's `stroke-width` prop instead of editing SVG paths. |
| Fill | Prefer outline icons. Use filled icons only when the state needs it and the component documents the behavior. |

## Icon Buttons

Icon-only buttons need an accessible name because the icon is decorative for assistive technology.

<Canvas of={Stories.IconButtonExample} />

```vue
<button type="button" aria-label="Search">
  <Search aria-hidden="true" :size="20" :stroke-width="2" />
</button>
```

## Accessibility

Decorative icons should use `aria-hidden="true"`. If an icon carries meaning without visible text, provide the meaning on the owning control with `aria-label`, visible text, or another accessible label.

Do not rely on color alone for status. Pair status icons with text such as "Error", "Warning", or "Success" when the state is important.
