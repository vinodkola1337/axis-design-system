# Icons

## Overview

Axis uses Lucide for product UI icons through the Axis `Icon` component. Lucide provides the SVG shapes; Axis tokens define how icons appear in the interface. Use the <a href={externalLinks.lucideIcons} target="_blank" rel="noreferrer">Lucide icon catalog</a> to choose icon names.

<Canvas of={Stories.OverviewExample} />

## Usage

```vue
<script setup lang="ts">
import { Search } from '@lucide/vue'
import { Icon } from '@vinodkola/axis-ui'
</script>

<template>
  <Icon :icon="Search" size="md" />
</template>
```

- Import only the icons you need from `@lucide/vue`.
- Render product icons with the Axis `Icon` component.
- Let icons inherit text color with `currentColor` unless the icon communicates a semantic state.
- Use the `color` prop for common standalone icon intent; each alias maps to an exported semantic color token.
- Use the `size` and `stroke` props instead of raw Lucide pixel props.
- Use one icon style in product UI. Do not mix Lucide, PrimeIcons, Material Symbols, and custom SVGs in the same surface unless the design system documents an exception.

## Token Behavior

Lucide icons should follow the surrounding component by default:

| Property | Axis rule |
|---|---|
| Color | Use `currentColor`, the Icon `color` prop, or a semantic color token such as `--axis-color-text-secondary`, `--axis-color-interactive-primary`, or `--axis-color-feedback-error`. |
| Size | Use the Axis `Icon` `size` prop. It maps to `--axis-icon-size-*` tokens. |
| Stroke | Use the Axis `Icon` `stroke` prop. It maps to `--axis-icon-stroke-width-*` tokens. |
| Fill | Prefer outline icons. Use filled icons only when the state needs it and the component documents the behavior. |

### Semantic Color

Icon uses `currentColor` by default, so color should usually come from the parent component. Use the `color` prop only when the icon carries standalone status or emphasis. These aliases resolve to existing exported color tokens; they do not introduce new color values.

| Color | Token |
|---|---|
| `inherit` | `currentColor` |
| `text` | `--axis-color-text-primary` |
| `text-secondary` | `--axis-color-text-secondary` |
| `text-disabled` | `--axis-color-text-disabled` |
| `text-inverse` | `--axis-color-text-inverse` |
| `interactive` | `--axis-color-interactive-primary` |
| `link` | `--axis-color-text-link` |
| `success` | `--axis-color-feedback-success` |
| `warning` | `--axis-color-feedback-warning` |
| `error` | `--axis-color-feedback-error` |

## API

The `icon` prop accepts imported Vue icon components from `@lucide/vue`. Use the <a href={externalLinks.lucideIcons} target="_blank" rel="noreferrer">Lucide icon catalog</a> to find the icon to import.

## Accessibility

Icon hides the internal SVG from assistive technology. Do not label icons that sit beside text or inside an accessible control; use `role="img"` with an accessible name only when the icon is meaningful on its own.

```vue
<Icon :icon="AlertTriangle" role="img" aria-label="Warning" />
```

Do not rely on color alone for status. Pair status icons with text such as "Error", "Warning", or "Success" when the state is important.
